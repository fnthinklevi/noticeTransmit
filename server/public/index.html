Security

把安全，做进每一层

通知不上云、管理后台二步验证、敏感数据加密——隐私与安全，从设计之初就被认真对待。

🔐

二步验证（TOTP）

管理后台登录启用 2FA，兼容 Google Authenticator，防止凭据泄露。

🔑

bcrypt 哈希

Token 使用 bcrypt 哈希验证，抵御暴力破解与彩虹表攻击。

🛡️

IP 智能封锁

10 分钟内验证码错误 3 次自动封锁 IP，阻断自动化爆破。

🔢

恢复码机制

生成 8 个一次性恢复码，设备丢失时安全找回账户。

🔒

AES-256-GCM 加密

TOTP secret 等敏感数据加密存储，密钥泄露也无法直接读取。

🛡️

通知不上云

所有通知、短信、通讯录仅在本地处理与推送，绝不上传任何服务器。推送只经你自己配置的 Webhook 发出——开发者不存储任何一条内容。

FAQ

常见问题

它会不会上传我的通知或短信内容？
不会。所有通知、短信、通讯录仅在本地监听、处理与推送，不会上传到任何服务器。推送只通过你自行配置的 Webhook 地址发出，开发者不存储任何推送内容。仅崩溃统计（腾讯 Bugly）会采集崩溃堆栈用于修复问题。
支持哪些推送平台？
内置适配企业微信群机器人、钉钉自定义机器人、飞书群机器人的消息格式；也支持任意自定义 Webhook 地址，可对接兼容的第三方平台或你自己的服务端。
为什么有时候收不到推送？
请依次检查：① 通知访问权限是否开启；② 是否已忽略电池优化（应用会在首次启用时引导）；③ 前台服务是否运行；④ 厂商自启动 / 后台权限是否放行；⑤ Webhook 地址是否正确（设置页可测试）；⑥ 是否被应用筛选或关键词过滤拦截。
需要 root 或特殊权限吗？
无需 root。仅需授予通知监听权限，并按需开启短信 / 来电识别、忽略电池优化、开机自启等常规权限即可稳定运行。
是免费的吗？开源吗？
完全免费，且在 GitHub 开源（MIT 许可，供学习交流使用）。欢迎提交 Issue 与 Pull Request 参与共建。
